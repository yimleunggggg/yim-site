# 通知（邮件 + 手机）

## 渠道

| 渠道 | Secret | 说明 |
|------|--------|------|
| GitHub Issue | 无 | Watch → Issues 收邮件 |
| ntfy | `NTFY_TOPIC` | 5–8 行纯文本摘要，无 Markdown `**` |
| Resend | `RESEND_API_KEY`、`SEO_NOTIFY_EMAIL`、`RESEND_FROM` | HTML 排版邮件 |

脚本：`scripts/seo_notify.sh`（日报/半月/周报 workflow 已调用）。

ntfy 摘要含：**结论 + 本周 P0 + 一条「为什么」+ 待办**（见 [06-日报与优化追踪.md](06-日报与优化追踪.md)）。

## Resend + DNSPod（域名发信）

1. [resend.com](https://resend.com) 创建 API Key
2. Resend 添加域名 → 按提示在 **DNSPod** 添加 SPF/DKIM/MX 记录
3. 验证通过后 `RESEND_FROM` 用 `Name <seo@你的域名>`
4. 测试：`workflow_dispatch` 跑 daily → 日志 `✓ email`

未配 Secret 时输出：`ℹ 未配置通知`，报告仍在 Git。

详见 [NOTIFICATIONS.md](../NOTIFICATIONS.md)。
