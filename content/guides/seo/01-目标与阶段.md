# 目标与阶段

## 要实现什么

| 能力 | 自动化效果 |
|------|------------|
| **日报** | 每天拉 GA4（昨日+7日）+ GSC（28天，滞后2–3天）+ 静态自检 → `reports/daily/` + 邮件/ntfy |
| **通知** | 可选 ntfy + Resend 邮件 |
| **不改站** | 日报只读数；执行优化在 Cursor 手动做 |
| **周报（可选）** | 本地 `seo_report_weekly.sh`，无定时 workflow |

屋久岛场景：工具站（时刻表/上岛交通），不堆旅游攻略；新站 GSC 展示 0 属正常节奏。

## 分几个阶段

```text
阶段 A（人工一次性）  GSC 验证 + GA4 埋点 + sitemap
        ↓
阶段 B（授权）        Cloud API + 服务账号 + OAuth + GitHub Secrets
        ↓
阶段 C（通知，可选）  Resend + DNSPod + ntfy
        ↓
阶段 D（云端跑）      Actions：seo-daily 每天
        ↓
阶段 E（你决策）      Cursor「跑一轮 SEO 优化」
```

## 不必依赖

- 本地电脑 24h 开机
- 自动开 GitHub Issue / approve 改站（已移除）
