## 1. 日报长什么样（看 ntfy 前先懂结构）

| 章节 | 内容 |
|------|------|
| **一、今日结论** | 一句话 + 数字表 + **本周优先（GSC/GA4 自动）** |
| **二、数据明细** | 指标释义、渠道/国家/着陆页（bot 行已剔除） |
| **三、为什么会这样** | 结合你的数据解释（Direct=微信、0 Organic 等） |

| **六、待办清单** | 从 P0 抽出的手动项 |
| **七、学一点** | 每天轮换一条概念 |

**不用手查 GSC 查询页**：`top_queries`、`index_status` 已在 Actions 里拉取并写入 §四。

生成脚本：`scripts/seo_daily_report.py` · 优先级：`scripts/seo_priorities.py`

## 2. 自动优先级怎么来的

| 数据源 | 会触发什么建议 |
|--------|----------------|
| `index_status` NEUTRAL | P0：URL 检查 → 请求编入索引 |
| `top_queries` 有展示无点击 | P0/P1：哪个词 → 应用哪条 URL（/map/、/access/） |
| GSC 28d 有展示 0 点击 | P0：排名/标题问题 |
| GA4 Direct 高 + 有人在用 | P1：先索引，不急改产品 |
| bot 国家行多 | P1：GA4 排除已知机器人 |

查询词 → 落地页规则在 `seo_priorities.py` 的 `_QUERY_ROUTES`（可改）。

## 3. 失败会自动重跑（与你电脑无关）

| 阶段 | 行为 |
|------|------|
| 第 1 轮 | cron 17:00 跑 `seo_daily_ci_with_retry.sh` |
| 第 2 轮 | 第 1 轮失败 → 等 **60 秒**再跑 |
| 每轮内 | GA4/GSC fetch、git push 各 **最多 3 次** |
| 两轮均失败 | ntfy **高优先级**：`日报失败 · 请人工处理` + Actions 链接 |

**常见误报**：ntfy 已收到但 Actions 红 X → 多为邮件 HTML 或 commit 步骤失败；重试/commit 修复后应变绿。

Workflow 单 job 实现：`scripts/seo_daily_ci.sh` + `seo_daily_ci_with_retry.sh`

### 站点不可用告警（2026-05）

Actions **第一步**跑 `seo_site_uptime.py` 探测 `/` 等四页：

- 非 **200** → 立即 ntfy **urgent**「站点不可用」（不等日报写完）
- 日报 §一 显示 ⚠ 线上站点异常 + P0「站点不可用」

**GSC 不会推送到手机**；只有你自己点「测试实际网址」才看到 Live 404。
