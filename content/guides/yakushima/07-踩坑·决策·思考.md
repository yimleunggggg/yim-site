# 踩坑 · 决策 · 思考

> 本项目真实遇到过的问题、为什么这样选、换一条路会怎样。  
> **社媒可摘「故事段」**；涉及真实邮箱的见私人日记。

---

## 1. Google Search Console：服务账号加不进去

**现象**：GSC「添加用户」填服务账号邮箱 → `email not found`。

**原因**：GSC 不认 Cloud 服务账号当普通 Google 用户（和 GA4 不同）。

**我们的解法**：OAuth — 用你的 Gmail 浏览器授权一次，refresh token 存 GitHub Secret，Actions 代你读 GSC。

**思考**：文档里凡写「SA 加 GSC」的旧教程，2024 年后很多站会踩坑。**先查官方当前支持的方式**，不要假设 SA 万能。

**Alternatives**：手动每月导出 GSC CSV（不可持续）；Search Console API + OAuth（我们选的）。

---

## 2. OAuth `access_denied` / `invalid_grant`

| 现象 | 原因 | 处理 |
|------|------|------|
| access_denied | OAuth 应用还在 Testing，你的 Gmail 不在 Test users | 同意屏幕 → Test users 加邮箱 |
| invalid_grant | 复制了占位符 token，或 token 过期 | 重跑 `seo_setup_gsc_oauth.py`，从 `secrets/gsc-oauth-token.json` 读真 token |

**思考**：Secrets 里一行都不能是 `你的token` 字面量。本地测通再进 Actions。

---

## 3. GA4 `403` / SERVICE_DISABLED

**现象**：Actions 拉 GA4 失败。

**处理**：

1. Google Cloud 启用 **Analytics Data API**  
2. `seo_grant_ga4_access.py` 把 SA 加为 GA4 **查看者**（属性 ID 是数字，不是 `G-xxx`）

**思考**：`G-xxx` 是前端埋点 ID；API 要用 **属性 ID**（Admin → 属性设置）。

---

## 4. 新站 GSC 展示 0、Organic 0

**不是 bug。**

**解释**：Google 还没把你排进搜索结果，或没人搜到你的词。工具站要时间 + 外链 + 持续收录。

**我们怎么做**：日报写「新站正常」+ 教看索引状态（PASS/NEUTRAL）+ 提醒 URL 检查 **手动** 请求收录。

**决策**：不加旅游攻略堆字数 — 会稀释「工具站」定位，和 GEO 策略一致。

---

## 5. 域名实名审核 vs GitHub Pages

**现象**：Custom domain 报 Invalid DNS。

**决策**：阶段 A 用 `github.io`，阶段 B 再绑 `yakushimabus.com` — **不阻塞开发**。

---

## 5b. Private 仓库 → 全站 404（2026-05）

**现象**：`yakushimabus.com` 显示 **Site not found · GitHub Pages**；GSC Live 404；SEO 日报/GSC API 仍正常（不反映站点挂了）。

**根因**：免费版 **Private 仓库不能托管 Pages**。改 Public **不会自动恢复** Pages。

**处理**：Public → Settings → Pages → `main` / root → Save → 自定义域 `yakushimabus.com`。

**排查**：`python3 scripts/seo_site_uptime.py`；`raw.githubusercontent.com/.../index.html` 若 200 而站点 404 = Pages 未发布。详 [deploy-yakushimabus.md](../deploy-yakushimabus.md) §6.1。

---

## 6. 邮件：Resend + DNSPod

**现象**：Resend 域名 Not Started。

**处理**：DNSPod 加 `resend._domainkey`、`send` MX/TXT、`_dmarc` — **不动** `@` 的 A 记录。

**验证后**：`RESEND_FROM = YakuBus SEO <seo@你的域名.com>` — **不需要**真开邮箱。

**测试期**：可用 `onboarding@resend.dev`，但只能发到 Resend 验证过的收件人。

---

## 7. ntfy 通了、邮件不通（或反过来）

**排查**：

- ntfy：Secret 主题名 = app 订阅名  
- 邮件：Resend Key、From 域名、垃圾箱  

**设计**：两个渠道独立，配一个也能用；`seo_notify.sh` 任一成功即可。

---

## 8. 本地跑 SEO 脚本 timeout

**现象**：国内网络拉 Google API 慢或失败。

**决策**：**以 GitHub Actions 为准**；本地只跑 `seo_report_daily.sh` 生成 Markdown（用已有 `latest.json`）。

---

## 9. Cursor 是否必须？

| 场景 | 要不要 Cursor |
|------|----------------|
| 日常看报告 | 不要 |
| 周报 `approve` | 不要（GitHub Issue 手机即可） |
| 大改 meta / 复杂 UI | 方便，用 Cursor 说人话 |
| 定时拉数发通知 | 不要 — Actions |

**思考**：把「决策」和「执行」分开 — 方案在 Issue，执行可以 bot PR。

---

## 10. AI 改太多 / 失忆

**坑**：一次对话改 20 个文件，下次 AI 不知道之前定了什么。

**解法**：

- 规则写进 `.cursor/rules` 或 playbook  
- 每阶段 `docs/notes/YYYY-MM-DD-*.md` 落盘  
- 对用户说「最小改动，只动 X」

---

## 11. PDF 解析错列 / 运价 silent wrong

**坑**：PDF 表格列错位 → 票价错但页面能打开。

**解法**：`build_all.py` **11 组锚点票价**校验，失败 exit 1；运价非官方逐站价标「目安」。

**思考**：数据类项目 **校验比 UI 更重要**。

---

## 12. 移动端 PDF 打不开

**决策**：地图页用 **预渲染 JPG 预览** + 可选 PDF 全屏，而不是强依赖手机 PDF 插件。

见 [map-pdf-mobile 笔记](../notes/2026-05-21-map-pdf-mobile-preview.md)。

---

## 13. 哪些文件不该 commit

| 别提交 | 原因 |
|--------|------|
| `secrets/` | 密钥 |
| `desktop-*.png` QA 截图 | 体积、临时 |
| `video/out/` | 可重新渲染 |
| 真实 token 在 md 里 | 泄露 |

用 `.gitignore` + `PRIVATE_SETUP.local.md`。
