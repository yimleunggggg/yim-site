# 部署与成本

## 托管：GitHub Pages

**是什么**：GitHub 把你的 HTML 当网站 host，免费。

**怎么做**（摘要）：

1. 代码在 `main` 分支  
2. Settings → Pages → Branch: `main` / root  
3. 等 1～2 分钟 → `https://用户名.github.io/仓库名/`

完整步骤：[deploy-yakushimabus.md](../deploy-yakushimabus.md)

---

## 自定义域名

| 步骤 | 说明 |
|------|------|
| 买域名 | 本项目：腾讯云 ¥83/首年 |
| DNS A 记录 ×4 | 指向 GitHub Pages IP（部署文档有列表） |
| 仓库 `CNAME` 文件 | 内容一行：`你的域名.com` |
| GitHub Pages 填域名 | Enforce HTTPS 勾选 |

**术语**：**DNS** 像电话簿，把域名翻译成服务器地址。**HTTPS** 是浏览器小锁，GitHub 免费签证书。

---

## 实名审核空窗期

腾讯云域名审核中：

- Pages 填自定义域名会 **InvalidDNSError** → 正常  
- 继续用 `github.io` 预览  
- 审核通过后再配 DNSPod  

我们当时就是这样，没有阻塞开发。

## 本地预览（不花钱）

```bash
cd 项目目录
python3 -m http.server 8765
# 浏览器打开 http://localhost:8765
```

改 CSS/文案时用；**SEO 拉 Google 数据**以 GitHub Actions 为准（国内本地常 timeout）。

---

## 发布节奏

```text
改代码 → git commit → git push origin main
         ↓
    GitHub Pages 自动更新（约 1～3 分钟）
         ↓
    可选：Actions 跑 SEO / 发通知
```

你不需要 FTP、不需要买服务器。
