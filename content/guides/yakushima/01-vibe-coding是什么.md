# Vibe Coding 是什么

## 先说结论

**Vibe coding** ≈ 你当产品经理 + 验收官，AI 当执行层写代码。  
你不需要会语法，但需要：

1. **说清楚要什么**（用户是谁、场景是什么）
2. **会看结果对不对**（打开网页、点一圈、对照官方 PDF）
3. **会做决定**（这个方案太复杂？换简单的；这个功能先不做）

---

## 和传统「学编程」有什么不同

| | 传统学编程 | Vibe coding |
|--|-----------|-------------|
| 起点 | 语法、框架 | 问题和用户场景 |
| 产出 | 自己写每一行 | AI 写，你改方向 |
| 失败时 | 查 Stack Overflow | 把报错贴给 AI，或换说法 |
| 适合 | 长期做工程师 | **快速做一个小而美的工具** |

本项目（屋久岛公交查询）作者背景：**不动代码**，用 Cursor 对话 + 少量复制命令，约 **1～2 个密集工作日** 做出可上线站点（详见 [PRIVATE_JOURNAL](PRIVATE_JOURNAL.md) 时间线）。

---

## 一个有效的对话节奏

```text
1. 定场景   「路边等公交，要查下一班几点」
2. 定边界   「只用官方 PDF，不做攻略社区」
3. 让 AI 做 MVP   先能查一条线
4. 你验收   手机打开，和 PDF 对时刻
5. 迭代     「站名要日文为主」「加英文 PDF」
6. 上线     推 GitHub → 绑域名
7. 自动化   「我想每天收到流量报告」→ 再加 Actions
```

**关键习惯**：每完成一阶段，让 AI **把结论写进 `docs/notes/` 或 playbook**，否则下次对话会「失忆」。

---

## 你需要准备的（很少）

| 项 | 用途 | 大概费用 |
|----|------|----------|
| Cursor（或同类 AI IDE） | 写代码、跑 Agent | 订阅制 |
| GitHub 账号 | 存代码 + 免费托管 | ¥0 |
| 域名（可选但推荐） | 像正式产品 | ~¥80/首年 |
| Google 账号 | GA4 + GSC | ¥0 |
| Resend + ntfy（可选） | 邮件 + 手机推送 | ¥0 起步 |

---

## 什么时候该警惕

- AI 一次改太多文件 → 说「**最小改动，只动 XX**」
- 方案出现后端、数据库 → 问「**能不能纯静态**」；小工具通常可以
- 密钥写进代码 → **永远放 GitHub Secrets / 本地 secrets/**
- Automating 一切 → 先手动跑通一次，再 cron
