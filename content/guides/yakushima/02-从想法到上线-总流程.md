# 从想法到上线：总流程

## 全景（五阶段）

```text
阶段 0  想法验证     「有人需要吗？官方数据公开吗？」
   ↓
阶段 1  能用的页面   本地能查时刻表，和 PDF 对得上
   ↓
阶段 2  完整产品     地图/运价/上岛/三语/about
   ↓
阶段 3  上线         GitHub Pages + 域名 + HTTPS
   ↓
阶段 4  被找到       GA4/GSC + SEO 自动化 + 通知
   ↓
阶段 5  持续运营     官方 PDF 换版 → 重建；周报 approve 改 meta
```

**原则**：先阶段 1 再 polish。很多 vibe coder 死在「界面还没能查车就开始研究 SEO」。

---

## 阶段 0：想法验证（几小时）

**本项目怎么定的：**

- 痛点：岛外旅客拿着 PDF 在路边翻时刻表
- 数据：屋久岛观光协会公开 PDF → **合法、可维护**
- 边界：**工具站**，不做游记、不做订票系统
- 名字 + 域名：`yakushimabus.com`（见 [cost-ledger](../notes/cost-ledger.md)）

**你要问自己的：**

1. 数据来源是否公开、能否写清楚出处？
2. 有没有已有 App 做得更好？你的差异是「更轻、更专、三语」吗？
3. 你愿意在 PDF 改点时更新吗？（通常一年 1～2 次）

---

## 阶段 1：MVP（第 1 天）

产出：`index.html` + `data.js`，手机能查「A 站 → B 站下一班」。

| 步骤 | 做什么 | 谁来做 |
|------|--------|--------|
| 下载官方 PDF | curl 或浏览器存到 `assets/` | 你或 AI |
| 解析 PDF | `scripts/parse_pdf.py` → 班次 JSON | AI |
| 生成前端数据 | `data.js` | AI |
| 做查询页 | 选站、筛时间、高亮下一班 | AI |
| **你验收** | 随机抽 3 班车和 PDF 核对 | **你** |

记录：[2026-05-20-init](../notes/2026-05-20-init.md)

## 阶段 2：产品化（第 1～2 天）

| 模块 | 页面 | 笔记 |
|------|------|------|
| 路线图 + 运价 | `/map/` | [map-pdf-mobile](../notes/2026-05-21-map-pdf-mobile-preview.md) |
| 上岛交通 | `/access/` | [access-booking](../notes/2026-05-21-access-booking.md) |
| 桌面布局 | 响应式 | [desktop-layout](../notes/2026-05-21-desktop-layout.md) |
| 关于页 | `/about/` | [product-intro](../product-intro.md) |
| 统一构建 | `build_all.py` | README |

**数据管线**（重要概念）：

```text
官方 PDF / JSON 源
    → Python 脚本（build_all.py）
    → data.js / map-data.js / access-data.js
    → 浏览器直接读，无需服务器
```

局部改时刻不用重下 PDF：用 `sources/overrides/`（见 [overrides README](../../sources/overrides/README.md)）。

## 阶段 3：上线

1. 代码 push 到 GitHub
2. Settings → Pages → `main` 分支
3. 先用 `用户名.github.io/仓库名` 预览
4. 买域名 → DNSPod 加 4 条 A 记录 → GitHub 绑自定义域名 → HTTPS

详步骤：[deploy-yakushimabus.md](../deploy-yakushimabus.md)

**我们踩过的坑**：域名实名审核期间 Pages 会报 DNS 错 → **正常，先忽略**，用 github.io 地址开发。

## 阶段 4：被找到（SEO 自动化）

目标：**不用每天打开 Google 后台**，手机收摘要，Git 里存完整报告。

| 任务 | 频率 | 文档 |
|------|------|------|
| 日报 | 每天 17:00 北京 | [seo/tutorial/03](../seo/tutorial/03-GitHub与Workflow.md) |
| 半月汇总 | 1/15 | 同上 |
| 周报方案 | 每周一 | [seo/tutorial/05](../seo/tutorial/05-报告与审批.md) |

**不必 Cursor、不必电脑开机** — Actions 在云端跑。

---

## 阶段 5：持续运营

| 触发 | 动作 |
|------|------|
| 官方发新 PDF | 下载 → 改 manifest → `python3 scripts/build_all.py` → push |
| 周报 Issue 你回复 `approve` | 改 title/meta（自动 PR 待接） |
| 日报异常飙涨/暴跌 | 看报告「洞察」段，是否部署或爬虫问题 |
| 想加功能 | 新对话 + 指向 playbook/notes 给 AI 上下文 |
