## 工具使用逻辑

<PullQuote>
任务导向 → 工具专长匹配 → 精准提问 → 工具组合协作 → 产出
</PullQuote>

<Callout type="心法">
每个 AI 工具背后使用的是不同模型或算法，在处理不同任务时存在能力差异。选择最擅长某类任务的工具，才能达到最佳效率与效果。
</Callout>

- **通用型大模型**（GPT 类）：逻辑理解 + 内容生成 + 信息整合 — ChatGPT、Claude、Gemini
- **专精型 AI 工具**：只做一件事，效率极高 — Midjourney（画图）、Perplexity（搜索）、Manus（文案）

## 工具选用

👉 **先看任务，而不是直接挑工具。**

不要上来就问：「我该用 GPT？Claude？Manus？哪个更强？」

这就像问：「我去旅行该穿登山鞋还是拖鞋？」你当然要先问清楚：你要去哪？山上还是沙滩？

同理，你是要：

- 翻译一个外文方案？
- 分析一批用户评论数据，做出 VOC 总结？
- 发一条海外社媒内容吸引点击？

不同的任务目标 → 要调用的能力差距很大。**任务导向，而不是工具导向。**

<PullQuote>
把 AI 当员工用之前，要像领导一样，把任务定清楚。
</PullQuote>

先写一句话问题：

- 我现在要完成什么？
- 最终输出形式是：文字 / 报告 / 图像 / 数据 / 视频？

再判断这个任务「偏重」哪一类：

- 信息收集类（需要联网）？
- 创意生成类（需要 AI 发挥想象）？
- 数据分析类（需要处理结构化内容）？
- 协作整理类（需要结构/文档）？

最后选工具：确定好「任务 + 输出形式」后再选工具，避免「工具先行」导致低效绕路。

## 组合方式

典型组合路径（课堂案例）：

1. **信息类**：秘塔 / Perplexity 搜索 → GPT 归纳 → Notion 成稿
2. **视觉类**：GPT 分镜 → MJ / Nano 出图 → 可灵 / Runway 视频 → Suno 配乐 → 剪映
3. **翻译类**：豆包翻译视频/文档 → GPT 润色 → 内部文档归档
4. **方案类**：Manus 搜集+结构 → GPT 细化 → PPT / 飞书

**组合原则**：上游输出格式 = 下游输入格式；避免让 GPT 直接「生成最终海报」一步到底。

## 提问方式

<Callout type="技巧" title="提问方式 · 课堂清单">
- **先任务后工具**：用一句话写清交付物
- **给反面例子**：「不要出现多余文字」「不要改变产品颜色」
- **分步拆解**：复杂任务拆成 3–5 步，每步一个 Prompt
- **让 AI 追问**：直到 95% 理解再执行
- **反向提示**：用成品倒推 Prompt 模板
- **角色切换**：同一对话里分别扮演「创意总监」「法务」「用户」审稿
</Callout>

## 平常用的多的工具

### Chat GPT

**1.1 GPT 模型的区别及应用场景**

| 类型 | 适合 | 注意 |
|------|------|------|
| 快速模型 | 草稿、头脑风暴、翻译 | 深度推理弱 |
| 推理/Pro 模型 | 方案、分析、复杂 Prompt | 慢、贵 |
| 多模态 | 读图、读 PDF | 注意隐私 |

**1.2 依据场景结合 Tools**

- **联网**：实时信息、竞品、新闻
- **代码解释器**：表格、数据、批量处理
- **图像输入**：评图、反推 Prompt、改稿意见
- **自定义 GPT**：固定 SOP 的任务（如「只写 Instagram 文案」）

**1.3 基础应用举例**

- 外文方案翻译 + 结构化摘要
- 用户评论 VOC 归纳（先贴数据，再给分析框架）
- 会议纪要 → 待办提取
- 分镜脚本 / 镜头表生成

### Manus — 写作 · 信息搜集 · 方案结构

适合需要「搜集 + 成稿 + 结构化」的一体化任务。

### 豆包 — 视频及文档翻译 & 总结

字节系，对中文视频/文档友好。

### Notion — 写作 · 文档 · 流程

协作整理与知识库沉淀。
